
```
**LL(1)?** Yes.
Edge: Fragile nullable tail; tiny changes break the disjoint predict sets.

### 6. Right-recursive sum with primed tail (LL(1))
```

S  ::= E S
S  ::= ε | “+” S
E  ::= “num” | “(” S “)”

```
**LL(1)?** Yes.
Edge: Classic left-recursion elimination example; deceptively simple but teaches tail handling.

### 7. EBNF-style sum with repetition (LL(1))
```

S ::= E { “+” E }
E ::= “num” | “(” S “)”

```
**LL(1)?** Yes.
Edge: Looks like a regex; repetition vs. end-of-input is a common edge for LL(1) checkers.

### 8. If-then with optional else (not LL(1))
```

Stat ::= “if” Exp “then” Stat MoreIf
MoreIf ::= “else” Stat “end” | ε

```
**LL(1)?** No (`else` appears in FOLLOW(Stat)).
Edge: Clean-looking optional that triggers FIRST/FOLLOW conflict.

### 9. Direct left-recursive list (not LL(1))
```

```
**LL(1)?** No (left recursion).
Edge: Extremely simple; every left-recursion detector should catch it instantly.

### 10. Precedence via layered repetition (LL(1))

Expr ::= Term { (”+” | “-”) Term }
Term ::= Factor { (”*” | “/”) Factor }
Factor ::= “num” | “(” Expr “)”

```
**LL(1)?** Yes.
Edge: Shared FIRST sets look dangerous but layering makes it LL(1); classic precedence test.

### 11. Floating-point literal (not LL(1))

Float ::= Fract [ Exp ] [ (“f” | “l” | “F” | “L”) ]

```
**LL(1)?** No (overlapping alternatives in Fract and optionals).
Edge: Regex-like but full of nullable paths and common prefixes.

### 12. Multiple nullables in sequence (LL(1))
